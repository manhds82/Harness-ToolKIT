Onboarding: từ git clone đến Control Portal

Hướng dẫn từng bước cho người mới — lấy source về, cài governance, và đưa số liệu lên Portal.

Bạn sẽ mất bao lâu? Khoảng 5–10 phút/project. Những chỗ cần thay giá trị của bạn được tô màu đỏ, ví dụ <TÊN-PROJECT>. Mọi lệnh đều copy-paste được.

1Lấy source về từ Git

Mở PowerShell, vào thư mục chứa code của bạn rồi clone:

# vào ổ/thư mục bạn muốn chứa project (vd E:\SourceCode)
cd E:\SourceCode

# clone project cần quản trị
git clone <URL-GIT-CUA-PROJECT>

# và clone bộ toolkit governance (chứa bundle + script cài)
git clone https://github.com/manhds82/Harness-ToolKIT.git
Toolkit dùng để làm gì? Repo Harness-ToolKIT chứa bundle governance (bộ hooks + script) và các script cài đặt. Bạn "cài" bundle này vào từng project để project có khả năng ghi telemetry và đẩy lên Portal.

2Tạo project trên Portal & lấy thông tin

a

Đăng nhập Portal → tạo project

Mở https://YOUR-PORTAL-HOST → trang Projects → + New Project → đặt tên (nên trùng tên thư mục cho dễ nhớ) → tạo.

b

Vào project → tab Settings → mục "Push telemetry"

Bạn cần 2 giá trị ở đây:

  • project_id — nằm trong đoạn snippet mẫu (copy được).
  • ingest key — bấm Reveal ingest key → Copy. Đây là "mật khẩu" để đẩy dữ liệu.
Bảo mật key Ingest key là bí mật cấp project. Không commit lên git, không gửi qua chat/email. Mỗi project có key riêng — không dùng chung. Lỡ lộ thì bấm Rotate key để đổi.

3Cài + cấu hình bằng 1 lệnh

Đây là cách nhanh nhất — 1 lệnh làm hết: cài hooks + ghi cấu hình + lưu key + test đẩy thử.

powershell -File E:\SourceCode\Harness-ToolKIT\setup-portal-sync.ps1 `
  -ProjectDir "E:\SourceCode\<TÊN-PROJECT>" `
  -ProjectId  "<PROJECT_ID>" `
  -IngestKey  "<KEY>" `
  -PortalUrl  "https://YOUR-PORTAL-HOST"

Script sẽ tự động:

  1. Cài bundle governance v1.2.1 (hooks + client push) vào project.
  2. Ghi .harness/portal-sync.json (portal_url + project_id).
  3. Ghi .harness/portal-sync.key (chỉ chứa key, đã tự gitignore).
  4. Test đẩy thử lên Portal ngay.
Kết quả mong đợi Dòng cuối in ra: [push-telemetry] OK: actions=… incidents=… usage=…. Nếu thấy actions=0 là bình thường — nghĩa là kết nối OK nhưng project chưa có dữ liệu (bạn chưa chạy phiên làm việc AI nào trong nó). Xem mục 5.

4Kiểm tra đã lên Portal

Mở Portal → trang Projects → tìm project của bạn. Các cột Tokens / Tokens 24h / Errors / Prompts / Blocked sẽ hiện số. Có thể bấm ⟳ Sync now để refresh, hoặc bấm tiêu đề cột để sắp xếp.

5Từ giờ về sau — hoàn toàn tự động

Ban lam viec voi Claude Code trong project │ ▼ (hooks tu ghi telemetry vao .harness/) ket thuc session │ ▼ hook SessionEnd tu chay push-telemetry ┌─────────────────────┐ │ Control Portal │ ← so lieu tu cap nhat └─────────────────────┘

Sau khi đã cấu hình, bạn không cần làm gì thêm: mỗi lần kết thúc một phiên Claude Code trong project, telemetry tự động đẩy lên Portal.

Muốn đẩy ngay không đợi kết thúc phiên:

powershell -File E:\SourceCode\<TÊN-PROJECT>\.harness\scripts\powershell\push-telemetry.ps1
Khuyến nghị: bật auto-push định kỳ (đáng tin cậy nhất) Hook lúc kết thúc phiên đôi khi không chạy (đóng cửa sổ đột ngột, Ctrl-C…). Để data luôn lên đều mà không phụ thuộc điều đó, cài một Windows Scheduled Task chạy push-all mỗi 15 phút — chạy 1 lần, cover mọi project: 
powershell -File E:\SourceCode\HarnessAI-ToolKIT\install-push-scheduler.local.ps1
Task tự đẩy telemetry của mọi project đã có key, bỏ qua project chưa có data. Xem log: Get-Content "$env:LOCALAPPDATA\HarnessPortal\push-all.log" -Tail 20. Gỡ: thêm -Uninstall.

6Cách thủ công (nếu không muốn dùng script tổng)

Nếu thích làm từng bước rõ ràng:

1

Cài bundle vào project

powershell -File E:\SourceCode\Harness-ToolKIT\install.ps1 `
  -BundleFile E:\SourceCode\Harness-ToolKIT\standard-governance-1.2.1.bundle.json `
  -TargetDir  "E:\SourceCode\<TÊN-PROJECT>" -Force -MergeClaude
2

Tạo file cấu hình

Tạo E:\SourceCode\<TÊN-PROJECT>\.harness\portal-sync.json:

{
  "portal_url": "https://YOUR-PORTAL-HOST",
  "project_id": "<PROJECT_ID>"
}
3

Dán key

Tạo …\.harness\portal-sync.key — toàn bộ nội dung file chỉ là cái key, đúng 1 dòng, không dấu ngoặc, không gì khác. Lưu lại.

4

Đẩy thử

powershell -File E:\SourceCode\<TÊN-PROJECT>\.harness\scripts\powershell\push-telemetry.ps1

7Troubleshooting

Hiện tượngNguyên nhân & cách xử lý
Push failed: 401 Key sai hoặc nhầm project. Vào Portal đúng project → Reveal ingest key → copy lại → dán lại vào portal-sync.key.
Push failed: 403 Tường lửa/CDN chặn. Client bản v1.2.1 đã gửi User-Agent riêng; đảm bảo bạn cài bản ≥ 1.2.1.
portal-sync.json not configured Chưa có file cấu hình. Chạy lại lệnh ở mục 3, hoặc tạo file theo mục 6.
OK: actions=0 hoài Chưa chạy Claude Code trong project → chưa có telemetry để đẩy. Cứ làm việc bình thường rồi đẩy lại.
Portal vẫn thấy 0 Bấm ⟳ Sync now trên trang Projects. Với project trên máy cá nhân, số chỉ lên sau khi push.
python not found khi chạy hook Cài Python 3 và tick "Add to PATH". Hook token-sampler cần Python để đọc transcript.

8Câu hỏi thường gặp

Project của tôi ở ổ E:\ (Windows) — có sync được không?

Được. Portal không đọc trực tiếp ổ E: của bạn; thay vào đó máy bạn đẩy (push) telemetry lên qua API. Đó chính là vai trò của portal-sync.json + portal-sync.key.

Nhiều project thì làm sao cho nhanh?

Làm lần lượt mỗi project 1 lần lệnh ở mục 3 (mỗi project 1 project_id + key riêng). Sau đó tất cả tự động.

Số liệu Tokens có phải hoá đơn chính xác không?

Không — đây là số liệu ước lượng đọc từ transcript của phiên làm việc (đủ tốt để theo dõi xu hướng), không phải hoá đơn billing.

Có cần chạy gì trên server không?

Không. Toàn bộ chạy trên máy của bạn. Server chỉ nhận dữ liệu bạn đẩy lên.
