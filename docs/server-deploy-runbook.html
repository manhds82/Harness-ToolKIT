Harness Portal — Runbook triển khai lên server

Cách deploy Portal (admin + member + backend + Postgres) lên một server Linux đang chạy nhiều hệ thống khác mà không làm hỏng chúng. Placeholders: root@YOUR_SERVER, portal.example.com, member.example.com. Author: Dau Sy Manh (manhds@) · Apache-2.0.

Nguyên tắc cô lập (áp dụng xuyên suốt): mọi thứ chạy trong một compose project riêng (network/volume/tên container riêng); container chỉ bind 127.0.0.1:<free-port>; không giành :80/:443; reverse-proxy chỉ thêm vhost mới, không sửa vhost cũ. Gỡ sạch = docker compose -p <project> down.

Điều kiện

  • Server Linux có Docker + docker compose, một reverse proxy (vd nginx) đang giữ :80/:443, và bạn ssh vào được.
  • 2 subdomain (hoặc 1) đã trỏ DNS về server: portal.example.com (admin), member.example.com (member).
  • Máy dev có PowerShell + OpenSSH client (để chạy script đưa source lên).

Các bước

  1. Đưa source lên server (nhập PAT 1 lần). Repo private → dùng script; PAT đi thẳng máy bạn → git credential store của server, không qua bên thứ ba: 
    powershell -File tools\deploy\fetch-source-to-server.ps1 -Server root@YOUR_SERVER -Username <gh-user>
# -> /opt/harness-portal (source), /opt/harness-portal/portal (compose)
  2. Sinh secrets trên server (không commit, không để lộ). Nếu tooling của bạn chặn lệnh chạm .env, đặt tên file khác và dùng --env-file: 
    cd /opt/harness-portal/portal
cat > deploy.vars <<EOF
POSTGRES_USER=harness_portal
POSTGRES_DB=harness_portal
POSTGRES_PASSWORD=$(openssl rand -hex 24)
PORTAL_MASTER_KEY=$(openssl rand -hex 32)
JWT_SECRET=$(openssl rand -hex 32)
ENVIRONMENT=production
EMAIL_BACKEND=console
KEY_PROVIDER=local
MEMBER_WEB_BASE_URL=https://member.example.com
ADMIN_VITE_API_BASE_URL=
MEMBER_VITE_API_BASE_URL=
EOF
chmod 600 deploy.vars
    Sao lưu PORTAL_MASTER_KEY vào vault. Mất nó = không giải mã được mọi provider key đã lưu.
  3. Chọn port trống + file override cô lập. Kiểm port đang dùng (ss -tlnH), chọn 2 port trống (vd 9600/9610). Tạo docker-compose.server.yml — không publish db/backend, web bind localhost: 
    services:
  postgres: { ports: !reset [] }
  backend:  { ports: !reset [] }
  gateway:  { ports: !reset [] }
  admin-web:  { ports: !override ["127.0.0.1:9600:80"] }
  member-web: { ports: !override ["127.0.0.1:9610:80"] }
    Dùng !override để thay port (không phải !reset [list], cái đó xoá sạch). Render kiểm trước khi chạy: docker compose --env-file deploy.vars -p harness-portal -f docker-compose.yml -f docker-compose.server.yml config | grep -E 'published|host_ip' — phải thấy đúng 127.0.0.1:9600/9610, không có gì khác.
  4. Build & chạy (project name riêng để cô lập tuyệt đối): 
    docker compose --env-file deploy.vars -p harness-portal \
  -f docker-compose.yml -f docker-compose.server.yml up -d --build
    Backend tự chạy alembic upgrade head + healthcheck; web/gateway chờ backend healthy.
  5. Thêm vhost reverse-proxy + TLS (chỉ thêm file mới, không sửa cũ). Tái dùng cert wildcard sẵn có nếu domain đã có (vd Cloudflare Origin cert phủ *.example.com), hoặc certbot: 
    # /etc/nginx/sites-enabled/portal.example.com.conf
server { listen 80; server_name portal.example.com; location / { return 301 https://$host$request_uri; } }
server {
  listen 443 ssl http2; server_name portal.example.com;
  ssl_certificate     /path/to/fullchain.pem;
  ssl_certificate_key /path/to/privkey.pem;
  location / { proxy_pass http://127.0.0.1:9600; proxy_set_header Host $host;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme; }
}
# (làm tương tự file member.example.com.conf -> 127.0.0.1:9610)
nginx -t && nginx -s reload   # test TRƯỚC, reload graceful (không rớt site khác)
    Web đã build same-origin (nginx trong container tự proxy /api → backend), nên vhost host chỉ cần forward toàn bộ về cổng web — không cần cấu hình CORS.
  6. Verify + kiểm không đụng ai: 
    docker compose -p harness-portal ps                       # 4 container up/healthy
curl -sk --resolve portal.example.com:443:127.0.0.1 https://portal.example.com/   # 200 (cert origin dùng -k)
curl -s https://portal.example.com/api/auth/oidc/status   # {"enabled":false} (proxy OK)
docker ps -q | wc -l                                      # tổng = (cũ) + 4, không mất cái nào
    Kiểm thêm vài site cũ vẫn trả 200.

Gỡ / rollback

docker compose -p harness-portal down            # gỡ container (giữ volume dữ liệu)
docker compose -p harness-portal down -v         # gỡ cả volume (XOÁ dữ liệu Portal)
rm /etc/nginx/sites-enabled/portal.example.com.conf /etc/nginx/sites-enabled/member.example.com.conf && nginx -t && nginx -s reload
Trung thực / an toàn (C10): cô lập theo compose-project nghĩa là build lỗi cũng không hỏng hệ khác; nhưng luôn render config để chắc port đúng trước khi up, và nginx -t trước khi reload. Không dùng overlay Caddy giành :80/:443 trên server đã có reverse proxy. Secrets sinh trên server, không commit.
