Bộ rule guard hiện tại — 30 deny-pattern

Guard đọc từ risk-policy.yaml (C2), khớp chỉ với lệnh shell (Bash/PowerShell) — không soi nội dung file Write/Edit nên không còn false-positive. Mỗi lần chặn ghi 1 security-event + 1 dòng ledger deny.

NhómChặn (ví dụ)
Xoá/format huỷ dữ liệurm -rf · del /F · Remove-Item -Recurse -Force · dd of=/dev/ · mkfs · wipefs/shred
Leo thang đặc quyềnsudo · Start-Process -Verb RunAs · chmod 777 · chown -R
Thực thi mã từ xa (RCE)download | shell · decode-to-shell · download | Invoke-Expression · -EncodedCommand
Phá VCS / lịch sửgit reset --hard · git push --force · git clean -f
Chống điều tra / tắt phòng thủhistory -c · xoá bash_history · tắt firewall/fail2ban · fork bomb
Rò rỉ bí mậtđọc id_rsa/.aws/credentials · pipe .env|credentials ra mạng

Đề xuất cải tiến hệ thống

Ưu tiên theo nguyên tắc “nâng tầng thấp nhất trước” và “chuyển chốt cứng lên gateway”.

H6 · AgentOpsƯu tiên cao
  • Anomaly alerting: cảnh báo khi token/cost 1 phiên vọt > 3× baseline.
  • Hallucination/error flag: thu tín hiệu retry-fail, judge REJECTED vào telemetry.
  • Drift: theo dõi thay đổi hành vi model theo thời gian.
H3 · EvaluationƯu tiên cao
  • Golden dataset theo dự án + regression-gate skill chạy trước deploy.
  • Gate cứng có thể enforce phía server (không chỉ skill).
  • Feedback test-fail → tự update spec/plan.
H5/H4 · EnforcementƯu tiên vừa
  • Approval workflow ở gateway cho action rủi ro cao (deploy/DB write).
  • Outbound allowlist enforce tại gateway (không chỉ deny-pattern cục bộ).
  • PII-in-log scanner trước khi ghi telemetry.
H2/H7 · Độ bềnƯu tiên vừa
  • Idempotency key cho tool side-effect (chạy 2 lần vẫn safe).
  • Fallback model ladder khi primary lỗi.
  • Transaction/rollback boundary cho workflow deploy.

Rule mới đề xuất cho guard NEW

Bổ sung vào risk-policy.yaml — bịt các lối tấn công phổ biến còn hở. Đều là data (C2), không sửa script.

Rule mớiChặn gìVì sao
Cloud metadata fetchtruy cập 169.254.169.254Đánh cắp credential IAM/instance (SSRF)
Reverse shellnc -e · bash -i >& /dev/tcp/Mở shell ngược ra ngoài
Docker privilegeddocker run --privileged · mount /var/run/docker.sockThoát container, chiếm host
Setuid escalationchmod u+s / +4000Tạo binary leo quyền
Rewrite lịch sử sharedgit filter-repo · push --mirrorXoá dấu vết trên nhánh chung

Đề xuất kèm cải tiến guard: thêm trường severity+id cho mỗi rule để Portal thống kê theo loại; và tách rule thành “deny cứng” vs “ask (cảnh báo, cho phép khi có lý do)”, tránh chặn nhầm khi thao tác hợp lệ.
