Nâng điểm CASAN của từng dự án — việc cần làm & prompt

Điểm mỗi dự án = 0.55 × baseline (năng lực bundle) + 0.45 × observed (bằng chứng thật của dự án). Baseline đã cao; muốn điểm dự án lên thì phải sinh ra bằng chứng. File này liệt kê chính xác việc cần làm + prompt để giao cho dự án tự làm.

Nguyên tắc

Portal chấm mỗi tầng H theo các "tín hiệu" quan sát được trong telemetry. Thiếu tín hiệu → điểm thấp. Có 3 loại việc:

  • PORTAL Admin bấm vài cái trên Portal (nhanh, hiệu lực ngay).
  • PROMPT Dán prompt vào Claude Code trong dự án để agent tạo bằng chứng (test, docs…).
  • TỰ ĐỘNG Chỉ cần dùng harness bình thường (chạy phiên) là có.
Sau khi làm, telemetry tự sync (task 15′ / SessionEnd / Sync now) → mở tab Harness Level điểm tự cập nhật. Không cần "đo" tay.

A. Việc trên Portal (admin) — ~5 phút/dự án, hiệu lực ngay

  • Budgets → mỗi dự án tạo 1 Token Budget (max tokens/cost + ngưỡng %). ⇒ H5 ↑ ngay (~+8).
  • Providers → thêm ≥2 provider connection (Tier-2 như Claude Code/Copilot/Ollama không cần key). ⇒ H2 & H7 ↑.
  • Source/Doc tab → chạy scan mapping (nếu backend đọc được source). ⇒ H1 ↑.
  • Sync now ở Projects list sau khi làm ⇒ điểm cập nhật ngay.

B. Prompt giao cho DỰ ÁN tự làm

Mở Claude Code trong thư mục dự án, dán từng prompt. Agent sẽ tạo ra bằng chứng (docs/test…) mà Portal chấm điểm.

① Prompt tổng hợp (dán 1 lần — nâng H1/H3/H7)

Dự án này dùng AI Harness (bộ standard-governance). Hãy giúp nâng độ trưởng thành CASAN:
1. Bổ sung tài liệu: tạo/cập nhật docs/SRS.md (yêu cầu) và docs/spec.md (kiến trúc & thiết kế) đủ để người mới hiểu dự án.
2. Kiểm thử: rà soát test hiện có; viết test cho các luồng chính còn thiếu; chạy toàn bộ và sửa đến khi XANH. Tóm tắt passed/failed theo từng suite.
3. Dùng nhiều subagent chạy SONG SONG để review code theo 3 góc: đúng đắn (correctness), bảo mật (security), hiệu năng (performance); tổng hợp phát hiện.
4. Tuyệt đối không đưa secret/API key vào code (dùng biến môi trường).
Cuối cùng liệt kê những gì đã làm và file nào đã tạo/sửa.

② Chỉ nâng H3 (Evaluation) — quan trọng nhất

Chạy toàn bộ test suite của dự án. Nếu thiếu test cho luồng chính, viết bổ sung.
Sửa mọi test fail đến khi tất cả XANH. In ra tổng kết dạng: suite | passed | failed | coverage.
Nếu dự án có pipeline verify/review, tạo evidence bundle cho thay đổi vừa rồi.

③ Chỉ nâng H1 (Context)

Viết tài liệu docs/SRS.md và docs/spec.md cho dự án: mục tiêu, phạm vi, kiến trúc, các module chính, luồng dữ liệu, cách chạy & deploy. Ngắn gọn, chính xác, để agent/người mới hiểu nhanh.
Sau đó liệt kê map: mỗi module/thư mục code ↔ phần tài liệu tương ứng.

④ Chỉ nâng H7 (Orchestration)

Dùng cơ chế subagent: chia việc review dự án thành các agent song song (bảo mật, hiệu năng, đúng đắn, test-coverage). Mỗi agent báo cáo riêng rồi tổng hợp. Mục tiêu tạo hoạt động đa-agent thực sự.
Lưu ý trung thực: điểm chỉ lên khi bằng chứng là thật (test xanh thật, docs thật). Đừng tạo test rỗng/docs giả — vừa vô nghĩa vừa sai tinh thần governance.

C. Quy trình đề xuất (làm 1 lượt cho mọi dự án)

  1. Portal: tạo Budget + thêm provider thứ 2 cho tất cả dự án (mục A).
  2. Với mỗi dự án lớn: mở Claude Code, dán Prompt ①, để agent tạo docs + test + review đa-agent.
  3. Chạy xong → Sync now (hoặc chờ task 15′).
  4. Mở tab Harness Level → chọn dự án → xem điểm đã lên, còn thiếu tín hiệu nào (dấu ○ xám trong scorecard) thì bổ sung tiếp.
Muốn siết chặt hơn: bật pdp_enforce: true (xem guide chính §11) để H4/H5/H3 enforce thật (chặn egress lạ, chờ duyệt deploy, chặn deploy khi test đỏ).

Checklist mỗi dự án

  • Đã tạo Token Budget (H5)
  • Đã có ≥2 provider connection (H2/H7)
  • Đã có docs/SRS.md + docs/spec.md (H1)
  • Test suite chạy XANH + có report (H3)
  • Đã chạy phiên có subagent (H7)
  • Đã Sync + kiểm tra tab Harness Level
