Hướng dẫn Data Sync lên Control Portal

Toolkit Harness AI · pipeline telemetry → Portal · cập nhật 2026-07-16

Tóm tắt 30 giây Hooks của harness tự ghi telemetry (token, tool call, lệnh bị chặn) vào .harness/ của mỗi project. Portal lấy data đó bằng 2 đường: (A) project nằm trên server → backend tự đọc file mỗi 5 phút; (B) project trên máy dev (Windows E:\…) → máy dev tự đẩy (push) lên API mỗi khi kết thúc session. Cả 2 đường đều idempotent — đẩy trùng không double-count.

1Kiến trúc tổng thể

Trung thực về giới hạn (nguyên tắc C10) Cột Tokens là số liệu best-effort đọc từ transcript của hooks (Tier-2 agentic), không phải số liệu metered tại proxy — dùng để theo dõi xu hướng, không phải hoá đơn. Token của một session chỉ được ghi khi session kết thúc (hoặc subagent dừng), nên số liệu trễ theo session, không realtime.

2Đường A — Project nằm trên server

Không cần làm gì cả. Điều kiện đủ (đã setup sẵn cho các project hiện có):

  • Project đã cài bundle standard-governance ≥ 1.2.0 (hooks bash mới).
  • Trường harness_root_ref của project trên Portal trỏ đúng path Linux (vd /opt/<project>).
  • Backend container mount thư mục chứa project ở chế độ read-only (/opt:/opt:ro trong compose overlay của server).

Backend tự ingest mỗi 5 phút (đổi qua env AUTO_INGEST_INTERVAL_SECONDS, 0 = tắt). Muốn sync ngay: trang Projects → nút ⟳ Sync now.

3Đường B — Project trên máy dev (Windows)

Backend không bao giờ đọc được ổ E:\ của bạn — thay vào đó máy dev tự đẩy telemetry lên qua API push. Setup một lần cho mỗi project, sau đó hoàn toàn tự động:

1

Tạo file config .harness/portal-sync.json

{
  "portal_url": "https://<portal-host>",
  "project_id": "<project-id>"
}

<project-id> lấy từ Portal: mở project → tab Settings → mục Push telemetry có sẵn snippet đúng để copy nguyên văn. File này đã được gitignore (chứa host nội bộ — không commit).

2

Lấy ingest key và lưu vào .harness/portal-sync.key

Portal → project → Settings → Push telemetry → Reveal ingest key → Copy → dán vào file .harness/portal-sync.key (đúng 1 dòng). File này đã gitignore — key không bao giờ được commit (nguyên tắc C5). Có thể dùng env HARNESS_PORTAL_INGEST_KEY thay cho file.

3

Xong — mỗi session tự đẩy

Hook SessionEnd tự chạy push-telemetry PS bash khi bạn đóng session Claude Code. Muốn đẩy tay bất kỳ lúc nào:

# Windows
powershell -File .harness\scripts\powershell\push-telemetry.ps1

# macOS / Linux
bash .harness/scripts/bash/push-telemetry.sh

Output mong đợi: [push-telemetry] OK: actions=… incidents=… usage=… tool_calls=…

Idempotent — đẩy lại thoải mái Client luôn gửi toàn bộ nội dung file; server dedupe theo hash (ledger entry hash, event_id, session_id+agent). Đẩy 10 lần cùng data → vẫn chỉ 1 bản ghi. Mất mạng lúc đẩy cũng không sao — lần đẩy sau tự bù đủ.

Project mới chưa có harness?

# cài bundle governance vào project (từ checkout toolkit)
powershell -File tools\harness-bundle\install.ps1 -TargetDir D:\path\to\your-project -MergeClaude
# (bash: tools/harness-bundle/install.sh --bundle <file> --target <dir> --merge-claude)

Sau đó tạo project trên Portal (lấy project-id) và làm 3 bước ở trên.

4API push (tham khảo kỹ thuật)

EndpointAuthBody
POST /api/ingest/{project_id} Header X-Ingest-Key (so sánh constant-time; không bao giờ đặt key trong URL) JSON: agentops, chain_jsonl, security_events, tool_calls — nội dung raw của từng file, tối đa 10 MB/file
GET /api/admin/projects/{id}/ingest-key JWT admin (member không bao giờ thấy key) —
POST /api/admin/projects/{id}/rotate-ingest-key JWT admin — (đổi key: mọi máy dev phải cập nhật lại file key)
POST /api/admin/projects/ingest-all JWT admin — (nút "Sync now" — chỉ ingest đường filesystem của org mình)

5Troubleshooting

Triệu chứngNguyên nhânCách xử lý
403 Forbidden khi push CDN/WAF (Cloudflare) chặn User-Agent lạ Client bản mới đã gửi UA harness-push-telemetry/1.0 — cập nhật script push (bundle ≥ 1.2.0). Nếu vẫn 403: kiểm tra WAF rule cho phép UA này.
401 Unauthorized Sai/thiếu ingest key, hoặc sai project_id Reveal lại key trên Portal, dán lại vào .harness/portal-sync.key. 401 đồng nghĩa transport đã thông — chỉ sai thông tin xác thực.
Push OK nhưng counts toàn 0 Data đã được ingest trước đó (dedupe), hoặc file telemetry local trống Bình thường. Chạy thêm session mới rồi push lại; hoặc kiểm tra .harness/telemetry/ có file chưa.
Cột stats trên Portal = 0 với project server harness_root_ref sai path, hoặc backend không mount được thư mục Settings → sửa path Linux đúng; kiểm tra compose overlay có mount :ro thư mục chứa project.
Tokens = 0 dù session chạy nhiều Session chưa kết thúc, hoặc sampler bản cũ (đọc field không tồn tại) Token chỉ ghi khi SessionEnd/SubagentStop. Cập nhật hooks bundle ≥ 1.2.0 (sampler mới đọc transcript).
Push từ máy dev bị treo lâu File telemetry quá lớn (>10MB) hoặc mạng chậm Server trả 413 nếu quá 10MB/file. Dọn bớt log cũ bằng rotate-logs script của harness.

6Bảo mật

  • Ingest key là secret cấp project — chỉ admin thấy; member không bao giờ thấy (không nằm trong API project thường).
  • Key chỉ đi qua header, không bao giờ qua URL/query (không lọt access log), so sánh constant-time chống timing attack.
  • File .harness/portal-sync.key và .harness/portal-sync.json đều đã gitignore — không bao giờ commit (C5).
  • Nghi ngờ lộ key → Rotate key trên Portal rồi cập nhật lại các máy dev.
  • Ledger chain.jsonl là hash-chain bất biến (C9) — server chỉ đọc, không bao giờ ghi ngược về project.
