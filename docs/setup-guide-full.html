1 Clone git về

Lấy toolkit về máy dev — trong đó có bộ cài (bundle) và mọi script.

# Bản public (ai cũng clone được)
cd E:\SourceCode
git clone https://github.com/manhds82/Harness-ToolKIT.git HarnessAI-ToolKIT

Sau khi clone, bundle mới nhất nằm ở:

E:\SourceCode\HarnessAI-ToolKIT\bundles\standard-governance\standard-governance-1.2.5.bundle.json
Ghi chú: repo này chỉ cần clone một lần. Nó không phải project của bạn — nó là “kho” chứa bộ cài để rải vào các project khác.

2 Cài bộ governance vào từng project

Dùng install.ps1 — nó verify hash rồi bung 92 file (hooks + scripts + policies + schemas) vào thư mục project của bạn, byte-exact.

powershell -File E:\SourceCode\HarnessAI-ToolKIT\tools\harness-bundle\install.ps1 `
  -BundleFile E:\SourceCode\HarnessAI-ToolKIT\bundles\standard-governance\standard-governance-1.2.5.bundle.json `
  -TargetDir  E:\SourceCode\<YourProject> `
  -Force -MergeClaude

Sau khi cài, project của bạn có thêm:

<YourProject>/ ├─ .claude/ # hooks (SessionStart/PreToolUse/…), agents, skills, settings.json ├─ .harness/ │ ├─ control/ # risk-policy.yaml (24 deny patterns), tool-registry, power-policy… │ ├─ scripts/ # push-telemetry.ps1, guard, sampler, ledger… │ ├─ schemas/ # JSON Schema cho mọi config (C8) │ └─ memory/ # constitution.md ├─ contracts/ # project / agent / tool-registry / workflow YAML └─ CLAUDE.md # đã merge phần governance (-MergeClaude), an toàn chạy lại
  • -Force: ghi đè file bundle cũ (an toàn — không đụng code app của bạn).
  • -MergeClaude: chèn phần governance vào CLAUDE.md; có sentinel chống merge trùng, chạy lại nhiều lần không sao.
Kết quả bước này: project đã có hooks + guard + push-telemetry. Từ giờ mỗi phiên Claude Code sẽ tự ghi telemetry. Chưa nối Portal — làm ở bước 3.

3 Tạo portal-sync.json & portal-sync.key

Hai file này nối project local ↔ project trên Portal. .json = địa chỉ; .key = chìa khoá. Không có script bí ẩn nào khác — chỉ 2 file này.

Cách A — Tạo thủ công (1 project)

Chạy trong thư mục project (đổi domain & project_id cho đúng):

# 3a. file cấu hình (không bí mật)
$proj = "E:\SourceCode\<YourProject>"
$cfg  = '{ "portal_url": "https://YOUR-PORTAL-DOMAIN", "project_id": "<PROJECT_ID>" }'
Set-Content -Path "$proj\.harness\portal-sync.json" -Value $cfg -Encoding utf8

# 3b. file key rỗng — sẽ dán key vào ở bước 5
New-Item -ItemType File -Path "$proj\.harness\portal-sync.key" -Force | Out-Null

Rồi mở .harness\portal-sync.key, dán ingest key lấy từ Portal (Project → Settings → Reveal ingest key), lưu lại. Xong.

Thay cho file key: có thể đặt biến môi trường HARNESS_PORTAL_INGEST_KEY — push-telemetry ưu tiên biến này trước file. Tiện cho CI, khỏi để key trên đĩa.

Cách B — Làm hàng loạt cho mọi project (khuyến nghị)

Nếu bạn có nhiều project, viết 1 script *.local.ps1 (đuôi .local.ps1 đã được gitignore) liệt kê project → id, cài bundle + ghi portal-sync.json một lượt, rồi dán key vào từng file .key. Mẫu:

$Portal = "https://YOUR-PORTAL-DOMAIN"
$Bundle = "E:\SourceCode\HarnessAI-ToolKIT\bundles\standard-governance\standard-governance-1.2.5.bundle.json"
$Install= "E:\SourceCode\HarnessAI-ToolKIT\tools\harness-bundle\install.ps1"

$map = @(
  @{ Dir="ProjectA"; Id="<PROJECT_ID_A>" },
  @{ Dir="ProjectB"; Id="<PROJECT_ID_B>" }
)
foreach ($p in $map) {
  $root = "E:\SourceCode\$($p.Dir)"
  if (!(Test-Path $root)) { continue }
  & $Install -BundleFile $Bundle -TargetDir $root -Force -MergeClaude | Out-Null
  $cfg = "{ ""portal_url"": ""$Portal"", ""project_id"": ""$($p.Id)"" }"
  Set-Content "$root\.harness\portal-sync.json" $cfg -Encoding utf8
  if (!(Test-Path "$root\.harness\portal-sync.key")) {
    New-Item -ItemType File "$root\.harness\portal-sync.key" -Force | Out-Null
  }
  Write-Host "OK $($p.Dir) — nhớ dán key vào .harness\portal-sync.key"
}
Lưu ý thứ tự: project_id chỉ có sau khi bạn tạo project trên Portal (bước 5). Nên trình tự thực tế: deploy Portal (bước 4) → tạo project lấy id + key (bước 5) → quay lại điền vào 2 file này.

4 Deploy Portal SERVER

Portal chạy bằng Docker: 1 backend (FastAPI) + 1 Postgres + 2 web (admin / member). Deploy trên server có Docker.

4.1 — Lấy source & cấu hình secret

git clone https://github.com/manhds82/Harness-ToolKIT.git harness-portal
cd harness-portal/portal
cp .env.example .env
# Sửa .env: đặt POSTGRES_PASSWORD, JWT_SECRET, KEK master key, SETUP_GATE_KEY…
# KHÔNG commit .env (C5)

4.2 — Chạy stack

# Local / thử nghiệm
docker compose up -d --build

# Production (Caddy TLS + không mở port trần + backup Postgres)
docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d --build

Migrations (Alembic) chạy tự động khi backend khởi động — không cần lệnh riêng. Kiểm tra:

docker compose ps          # 4 service Up
docker compose logs -f backend

4.3 — Khởi tạo lần đầu (bootstrap)

Mở admin UI trên trình duyệt → màn Login yêu cầu Setup gate key để tạo Organization admin đầu tiên (chống người lạ bootstrap). Nhập key đã đặt trong SETUP_GATE_KEY. Sai 5 lần / IP → khoá 8 giờ.

Prod: đặt sau reverse-proxy TLS (Caddyfile có sẵn, tự cấp HTTPS). Secret nạp qua file *_FILE. Chi tiết vận hành xem server-deploy-runbook.html.

5 Add project & setting trên Portal

Làm trên Admin UI. Đây là nơi sinh ra project_id và ingest key để điền vào bước 3.

  1. Đăng nhập admin (tick Remember me = giữ 7 ngày).
  2. Projects → New project: đặt tên. Xong sẽ có 1 dòng project — copy Project ID.
  3. Vào project → tab Settings → Reveal ingest key → copy ingest key (có nút Rotate nếu cần đổi).
    Đây là 2 giá trị điền vào bước 3: Project ID → portal-sync.json; ingest key → portal-sync.key.
  4. Members: add theo email → sinh invite link cho member tự nhận.
  5. Token Budgets: đặt max tokens / max cost / ngưỡng cảnh báo (%).
  6. Provider Connections: chọn provider.
    TierProviderAPI key?Cách đo token
    Tier 1 (metered)Anthropic API, OpenAI, Deepseek, Qwen, GLM✅ Bắt buộc (mã hoá khi lưu)Proxy thật → đếm chính xác
    Tier 2 (agentic)Claude Code, GitHub Copilot, Codex, Ollama❌ Không cầnBest-effort từ transcript/hook
Kết quả: project đã tồn tại trên Portal, có budget + provider + members. Giờ quay lại bước 3 điền id + key vào máy dev.

7 Lệnh Sync data (token / prompt / security) lên Portal

Có 3 cách, tăng dần độ tiện. Tất cả đều idempotent (chạy lại nhiều lần không nhân đôi — server tự dedupe).

7.1 — Push 1 project (thủ công)

powershell -File E:\SourceCode\<YourProject>\.harness\scripts\powershell\push-telemetry.ps1

Trước khi push, script tự resample transcript của chính project đó (dựng lại agentops.log từ log Claude Code) — nên token được tính đủ kể cả khi hook SessionEnd không chạy.

7.2 — Sync TẤT CẢ project một lượt (khuyến nghị)

powershell -File E:\SourceCode\HarnessAI-ToolKIT\harness-sync.local.ps1

Script này tự quét mọi thư mục dưới E:\SourceCode\* có đủ portal-sync.json + portal-sync.key, resample transcript (khớp không phân biệt hoa/thường ổ đĩa, đệ quy cả subagents), rồi push từng project. Thêm project mới không phải sửa script.

7.3 — Tự động mỗi 15 phút (Scheduled Task)

# cài 1 lần — chạy nền, không phụ thuộc bạn có đóng Claude hay không
powershell -File E:\SourceCode\HarnessAI-ToolKIT\install-push-scheduler.local.ps1

# đổi chu kỳ / gỡ
powershell -File E:\SourceCode\HarnessAI-ToolKIT\install-push-scheduler.local.ps1 -IntervalMinutes 10
powershell -File E:\SourceCode\HarnessAI-ToolKIT\install-push-scheduler.local.ps1 -Uninstall
Ngoài ra: mỗi khi một phiên Claude Code kết thúc, hook SessionEnd cũng tự gọi push-telemetry. Vậy có 3 lớp: hook cuối phiên + scheduled task + chạy tay khi cần.

8 Verify & hiểu con số

Mở Admin → Projects list. Mỗi chỉ số hiển thị theo Hôm nay / Tháng này / Tất cả.

  • Cột Tokens, Cost: số nhóm bằng dấu phẩy hàng nghìn; tiền làm tròn đô-la.
  • Cột Errors / Prompts / Blocked: bấm vào số > 0 → nhảy đúng tab chi tiết trong project.
  • Last sync cập nhật sau mỗi lần push. Bấm Sync now trên Portal để ingest lại phía server.
“Sync 31d ago” không phải lỗi nếu project đó thực sự lâu không chạy — token đến từ transcript thật, tool không bịa dữ liệu. Chạy lại project bằng Claude Code → phiên mới sẽ được sync lên.

Kiểm tra nhanh 1 project vừa push:

powershell -File E:\SourceCode\HarnessAI-ToolKIT\harness-sync.local.ps1
# đọc dòng "OK <project> sessions=.. usage=.. actions=.." trả về từ server

➕ Thêm một project mới (checklist)

Khi bạn có project mới cần đưa lên Portal — làm đúng 4 bước, ~2 phút.

  1. Trên Portal (Admin): Projects → New project → copy Project ID + ingest key (Settings → Reveal).
  2. Cài bundle vào thư mục project: 
    powershell -File E:\SourceCode\HarnessAI-ToolKIT\tools\harness-bundle\install.ps1 `
  -BundleFile E:\SourceCode\HarnessAI-ToolKIT\bundles\standard-governance\standard-governance-1.2.5.bundle.json `
  -TargetDir E:\SourceCode\<NewProject> -Force -MergeClaude
  3. Tạo 2 file sync: 
    $proj = "E:\SourceCode\<NewProject>"
Set-Content "$proj\.harness\portal-sync.json" `
  '{ "portal_url": "https://YOUR-PORTAL-DOMAIN", "project_id": "<NEW_PROJECT_ID>" }' -Encoding utf8
New-Item -ItemType File "$proj\.harness\portal-sync.key" -Force | Out-Null
# rồi mở portal-sync.key, DÁN ingest key, lưu
  4. Push lần đầu (hoặc chờ scheduled task tự chạy): 
    powershell -File "$proj\.harness\scripts\powershell\push-telemetry.ps1"
Không cần sửa harness-sync.local.ps1: lần chạy sau nó tự phát hiện project mới nhờ 2 file portal-sync.*. Scheduled task cũng tự cuốn theo. Không lặp lại lỗi mất sync.
