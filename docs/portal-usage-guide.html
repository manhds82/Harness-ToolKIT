Harness Portal — Hướng dẫn sử dụng

Setup lần đầu → tạo project → gán budget → connect provider → đo token; và cách đưa một dự án đang chạy local vào quản lý tập trung. Placeholders: admin https://portal.example.com, member https://member.example.com. Author: Dau Sy Manh (manhds@) · Apache-2.0.

Portal cho bạn gì: tạo Project → thêm Member → gán Token Budget → connect AI provider (BYOK, key mã hoá) → đo token & chi phí + dashboard (bug/findings, prompt bị chặn, vi phạm, làm-lại). Điều kiện: Portal đã được deploy (xem runbook triển khai).

Phần 1 — Vận hành Portal

  1. First-time setup (Org Admin). Mở admin UI https://portal.example.com → tab First-time setup → tạo Organization + tài khoản Org Admin (bạn tự đặt email + mật khẩu mạnh). Đăng nhập.
  2. Tạo Project. Ví dụ acme-web. Mỗi project là ranh giới quản lý (member, budget, provider, log riêng).
  3. Gán Token Budget. Đặt max_tokens_per_session, max_cost_per_session_usd, alert_at_percent (vd 80%). Có thể đặt theo cả project lẫn từng member.
  4. Connect Provider (BYOK). Tab Providers → chọn provider (Anthropic API / OpenAI / Deepseek / Qwen / GLM = Tier-1 đo token thật; Claude Code / Copilot / Codex = Tier-2 best-effort) → dán API key. Key được mã hoá khi lưu (đọc DB chỉ thấy ciphertext, API luôn mask).
  5. Đo token. Với connection Tier-1: gửi request qua proxy của Portal — mỗi lần đo chính xác tokens_in/out + chi phí, cộng vào budget, cảnh báo ở ngưỡng, chặn 402 khi cạn. Dùng panel “Try it” trong member UI, hoặc gọi API: 
    POST https://portal.example.com/api/member/projects/{project_id}/proxy/chat
{ "provider_connection_id":"...", "model":"...", "messages":[{"role":"user","content":"hello"}] }
  6. Thêm Member. Add theo email → Portal sinh invite link (hoặc gửi email nếu bật SMTP) → member mở https://member.example.com/accept-invite?token=... đặt mật khẩu → chỉ thấy project được add, budget/provider (key mask).
  7. Theo dõi. Dashboard: cost/token theo member+provider, budgets, action log, security incidents, prompt scores, test reports (bấm ingest/scan/run để cập nhật từ checkout).

Phần 2 — Đưa một dự án đang chạy local vào quản lý

Kịch bản: bạn đã có một dự án chạy local (đã dùng nhiều), giờ muốn quản trị nó qua Portal như các service khác trên máy. Harness chạy ngay trong repo dự án (không đổi cách bạn chạy app); Portal đọc lại log/telemetry của repo đó để hiển thị tập trung.

  1. Cài governance bundle vào repo dự án (một lần): 
    # Windows
powershell -File install.ps1 -BundleFile standard-governance-1.0.0.bundle.json -TargetDir C:\path\to\my-service
# macOS/Linux
bash install.sh --bundle standard-governance-1.0.0.bundle.json --target /path/to/my-service
    Kiểm đã áp đúng: harness-verify.ps1 -TargetDir C:\path\to\my-service → APPLIED OK. Khởi động lại Claude Code trong repo để hook có hiệu lực.
  2. Đăng ký dự án trên Portal (tab Settings của project): đặt
    • harness_root_ref = đường dẫn checkout thật của dự án (để Portal ingest log/telemetry của nó).
    • gateway_base_url = URL gateway phục vụ checkout đó (nếu dùng đồng bộ qua gateway).
  3. Ingest lịch sử vào Portal. Bấm Run ingest/scan/run ở các tab → Portal đọc .harness/ledger/chain.jsonl + .harness/telemetry/* của repo → hiển thị: action log, security incidents (prompt bị chặn/vi phạm), prompt scores, source↔doc mapping, test report.
  4. Gán budget + connect provider cho project (như Phần 1) nếu muốn đo/chặn chi phí.
  5. Lặp cho từng dự án local khác. Mỗi dự án = 1 project trong Portal; bạn quản lý tập trung nhiều dự án cùng lúc, mỗi cái vẫn chạy độc lập trên máy.
Trung thực về đo token (C10): Portal đo chính xác chỉ với request gửi qua proxy Tier-1 của Portal. Token mà Claude Code tiêu khi bạn code trong dự án là Tier-2 best-effort — Portal chỉ ingest xấp xỉ từ ledger/telemetry, không đo tuyệt đối được (giới hạn kỹ thuật, không hứa suông). Muốn con số chi phí chính xác cho một luồng AI → cho luồng đó gọi model qua proxy Tier-1.
